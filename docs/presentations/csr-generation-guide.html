OpenSSL 다운로드 및 설치

Windows용 OpenSSL 바이너리를 다운로드하고 설치합니다

Step 1-1

OpenSSL 다운로드

아래 사이트에서 Windows용 OpenSSL 설치 파일을 다운로드합니다.

다운로드 사이트
https://slproweb.com/products/Win32OpenSSL.html

다운로드 목록에서 아래 버전을 선택합니다:

Win64 OpenSSL v3.x.x Light ← 이 버전을 다운로드 (EXE 설치 파일)
Light 버전으로 충분합니다. Full 버전(~300MB)은 개발용이므로 불필요합니다.
Step 1-2

설치 실행

다운로드한 EXE 파일을 관리자 권한으로 실행합니다.

  1. 설치 경로: C:\Program Files\OpenSSL-Win64 (기본값 유지)
  2. DLL 복사 옵션에서 아래를 선택합니다:
☑ Copy OpenSSL DLLs to: The Windows system directory

나머지 옵션은 기본값으로 진행 후 Install 버튼을 클릭합니다.

2

환경변수 설정 및 설치 확인

시스템 PATH에 OpenSSL을 추가하고 정상 설치를 확인합니다

Step 2-1

환경변수 PATH 추가

어느 경로에서든 openssl 명령을 사용할 수 있도록 시스템 PATH에 추가합니다.

  1. Win + S → "환경 변수" 검색 → "시스템 환경 변수 편집" 클릭
  2. "환경 변수(N)..." 버튼 클릭
  3. 시스템 변수 목록에서 Path 선택 → "편집(I)..." 클릭
  4. "새로 만들기(N)" 클릭 후 아래 경로 입력:
C:\Program Files\OpenSSL-Win64\bin

"확인" 버튼을 차례로 클릭하여 저장합니다.

Step 2-2

설치 확인

새 CMD 창을 열고 아래 명령을 실행합니다. (기존 CMD 창은 환경변수가 반영되지 않습니다)

CMD openssl version
OpenSSL 3.4.1 11 Feb 2025 (Library: OpenSSL 3.4.1 11 Feb 2025)
위와 같이 버전 정보가 출력되면 설치가 정상적으로 완료된 것입니다.
'openssl'은(는) 내부 또는 외부 명령...이 아닙니다 오류 시 환경변수 PATH 설정을 재확인하거나 PC를 재시작하세요.
3

CSR PEM 파일 생성

RSA 2048비트 키 쌍과 CSR을 한 번에 생성합니다

Step 3-1

명령 프롬프트 실행

Win + R → cmd 입력 → Enter

작업 폴더(파일이 저장될 위치)로 이동합니다:

CMD cd %USERPROFILE%\Desktop
Step 3-2

CSR + 개인키 생성

아래 명령을 복사하여 실행합니다. 한 줄로 입력하세요:

CMD openssl req -new -newkey rsa:2048 -nodes -keyout private.key -out csr.pem -subj "/C=KR/O=Ministry of Justice/CN=Korea Immigration Service"
명령 실행 후 바탕화면에 2개의 파일이 생성됩니다.
CSR
csr.pem
CSR 파일 — ICAO에 제출하는 파일
KEY
private.key
개인키 — 절대 외부 공유 금지
참고

명령어 옵션 설명

옵션설명
-new새 CSR 요청 생성
-newkey rsa:2048RSA 2048비트 키 쌍 신규 생성 (ICAO 필수 요구사항)
-nodes개인키에 암호를 설정하지 않음 (FASTpass® SPKD Import 편의)
-keyout private.key개인키 저장 파일명
-out csr.pemCSR PEM 파일 저장 파일명
-subj "..."인증서 주체 정보 (Subject Distinguished Name). ICAO는 제한 없음 — 자유 설정 가능
4

생성 결과 검증

CSR 파일과 개인키가 올바르게 생성되었는지 확인합니다

검증 1

CSR 내용 확인

CSR 파일의 내용을 사람이 읽을 수 있는 형태로 출력하고 서명을 검증합니다.

CMD openssl req -in csr.pem -text -noout -verify
Certificate request self-signature verify OK Certificate Request: Data: Version: 1 (0x0) Subject: C = KR, O = Ministry of Justice, CN = Korea Immigration Service Subject Public Key Info: Public Key Algorithm: rsaEncryption Public-Key: (2048 bit) ← RSA 2048 확인 ... Signature Algorithm: sha256WithRSAEncryption ← SHA256 확인

확인해야 할 3가지:

  • verify OK — CSR 서명이 유효합니다
  • Public-Key: (2048 bit) — RSA 2048비트 키 (ICAO 요구사항)
  • sha256WithRSAEncryption — SHA256withRSA 서명 (ICAO 요구사항)
검증 2

개인키 유효성 확인

CMD openssl rsa -in private.key -check -noout
RSA key ok
RSA key ok가 표시되면 개인키가 정상입니다.
검증 3

공개키 매칭 확인 (CSR ↔ 개인키)

CSR에 포함된 공개키와 개인키에서 추출한 공개키가 동일한지 확인합니다.

CMD — CSR 공개키 해시 openssl req -in csr.pem -noout -pubkey | openssl md5
CMD — 개인키 공개키 해시 openssl rsa -in private.key -pubout | openssl md5
CSR 공개키: MD5(stdin)= a1b2c3d4e5f6... 개인키: MD5(stdin)= a1b2c3d4e5f6... ← 동일하면 정상!
두 명령의 MD5 해시값이 동일하면 CSR과 개인키가 정상적으로 쌍을 이루고 있습니다.
검증 4

PEM 형식 육안 확인

CSR 파일 내용을 직접 열어 PEM 형식인지 확인합니다.

CMD type csr.pem
-----BEGIN CERTIFICATE REQUEST----- MIICjTCCAXUCAQAwSDELMAkGA1UEBhMCS1IxHDAaBgNVBAoME01pbmlz dHJ5IG9mIEp1c3RpY2UxGzAZBgNVBAMMEktvcmVhIEltbWlncmF0aW9u ... (Base64 인코딩된 데이터) ... -----END CERTIFICATE REQUEST-----
  • -----BEGIN CERTIFICATE REQUEST-----로 시작
  • 중간에 Base64 인코딩 문자만 존재 (알파벳, 숫자, +, /, =)
  • -----END CERTIFICATE REQUEST-----로 종료
5

ICAO PKD 포털에 CSR 제출

생성된 CSR PEM 파일을 ICAO에 제출합니다

Step 5-1

CSR 제출

  1. ICAO PKD 포털에 접속
  2. csr.pem 파일을 업로드하여 CSR 제출
  3. ICAO가 CSR을 검증하고 인증서를 발급합니다 (수일 소요)
Step 5-2

발급된 인증서 수령

  1. ICAO로부터 발급된 인증서 파일을 다운로드
  2. 인증서 파일과 개인키(private.key)를 안전하게 보관
발급된 인증서는 향후 사업 수행 시 FASTpass® SPKD 시스템에 등록하여 사용합니다. 자세한 절차는 부록 — FASTpass® SPKD Import 절차를 참고하세요.

부록

Subject DN 예시, 보안 주의사항, FASTpass® SPKD Import 절차

참고

Subject DN 예시

ICAO는 Subject DN에 제한을 두지 않습니다. 기관에 맞게 자유롭게 설정할 수 있습니다.

용도Subject DN
기본 /C=KR/O=Ministry of Justice/CN=Korea Immigration Service
부서 포함 /C=KR/O=Ministry of Justice/OU=Immigration/CN=Korea PKD
영문 약칭 /C=KR/O=MOJ/CN=KIS PKD Client

C=국가코드, O=조직명, OU=부서명, CN=일반이름

보안 주의

보안 주의사항

private.key 파일을 절대 외부에 공유하지 마세요
개인키가 유출되면 인증서의 보안이 완전히 무력화됩니다.
ICAO에는 CSR 파일(csr.pem)만 제출합니다
개인키(private.key)는 절대 ICAO에 제출하지 않습니다.
개인키를 안전한 장소에 백업하세요
개인키를 분실하면 발급된 인증서를 사용할 수 없습니다. USB 등 오프라인 매체에 백업을 권장합니다.

전체 명령어 요약

단계명령어
CSR 생성 openssl req -new -newkey rsa:2048 -nodes -keyout private.key -out csr.pem -subj "/C=KR/O=Ministry of Justice/CN=Korea Immigration Service"

개인키 검증 openssl rsa -in private.key -check -noout

openssl rsa -in private.key -pubout | openssl md5

FASTpass® SPKD 시스템 CSR Import 절차

아래 절차는 향후 사업 수행 시 FASTpass® SPKD 시스템이 구축된 후 진행합니다. CSR 생성 및 ICAO 제출은 사업 수행 전에도 가능합니다.

CSR + 개인키 Import:

  1. FASTpass® SPKD에 관리자 계정으로 로그인
  2. 좌측 메뉴 시스템 관리 → CSR 관리 클릭
  3. 상단 "CSR Import" 버튼 클릭
  4. CSR 파일: csr.pem 선택
  5. 개인키 파일: private.key 선택
  6. "Import" 버튼 클릭
자동 검증 항목 — Import 시 시스템이 자동으로 검증합니다:
  • CSR 서명 유효성 (X509_REQ_verify)
  • 개인키 ↔ CSR 공개키 매칭 (EVP_PKEY_eq)

ICAO 발급 인증서 등록:

  1. FASTpass® SPKD → CSR 관리 → 해당 CSR 선택
  2. "인증서 등록" 버튼 클릭
  3. ICAO로부터 발급받은 인증서 파일 선택 후 등록
인증서 등록 시 CSR의 공개키와 인증서의 공개키가 자동으로 매칭 검증됩니다. 개인키는 AES-256-GCM으로 암호화되어 안전하게 저장됩니다.
