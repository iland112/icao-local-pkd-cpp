### 현행 전자여권시스템 현황과 문제점

#### 현행 운영 현황

- ✓ ICRM 기반 입·출국심사 시 전자여권 판독·인증
- ✓ BAC/CA/AA 인증 + PKI Toolkit PA 검증
- ✓ 자동출입국심사(SES) 18개국 운영
- ✓ PKD Manager로 ICAO PKD → Local PKD 동기

#### 핵심 문제점

- ✗ **시스템 노후화** — PKD Manager 2009년 도입, HW/SW 6년+ 내용연수 초과 및 지원 만료
- ✗ **기술 표준 미지원** — LDS 1.8, PACE 미지원 (ICAO 2028년 의무 적용 권고 중)
- ✗ **ML 미활용** — ICAO Master List을 통한 CSCA 인증서 보조 수집·검증 부재
- ✗ **DL·Non-Conformant 미활용** — 각국 DL 및 비표준 DSC 미준수 사항 관리·조회 부재

#### 추가 개선 필요 사항

| 영역 | 문제 |

### Passive Authentication 검증 체계 도입의 필요성

#### ICAO 기준 변화

- **2010년대 (ICAO 권고)**: "Passive Authentication is **recommended**"
- **2024년 (ICAO Doc 9303 개정)**: "Passive Authentication **MUST** be implemented"

#### PA 검증이 보장하는 것

- ✓ **칩 데이터 무결성** — SOD 서명으로 변조 탐지
- ✓ **발급국 인증** — CSCA → DSC Trust Chain 검증
- ✓ **인증서 유효성** — CRL 기반 폐기 여부 확인
- ✓ **DG 해시 무결성** — 사진(DG2)·MRZ(DG1) 등 원본 보장

#### PA 검증 프로세스

| 단계 | 내용 |
|------|------|
| Step 1-2 | SOD 서명 검증 + DSC 조회 |
| Step 3-4 | CSCA 조회 + Trust Chain 검증 |
| Step 5-7 | SOD 해시 추출 + DG 해시 비교 |
| Step 8 | CRL 폐기 여부 확인 |

- **검증 정확도**: 99.9%
- **처리 시간**: 100-300ms
- **자동 검증**: 8단계
