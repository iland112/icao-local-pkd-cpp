## 1. 개요

ICAO PKD LDAP 서버에 TLS(암호화) 연결로 인증서를 자동 동기화하려면 **클라이언트 인증서**가 필요합니다. 이 매뉴얼은 CSR 생성부터 TLS 연결 확인까지 Frontend에서 직접 수행하는 전체 절차를 안내합니다.

### 전체 흐름

```
┌─────────────────────────────────────────────────────────────────────┐
│                        CSR 관리 페이지                               │
│                                                                     │
│  Step 1                  Step 2                    Step 3           │
│  ┌──────────────┐       ┌──────────────────┐      ┌──────────────┐ │
│  │  CSR 생성     │  ──→  │ TLS 클라이언트    │  ──→ │ ICAO PKD     │ │
│  │  (서명 요청서) │       │ 인증서 발급       │      │ 연결 적용     │ │
│  └──────────────┘       └──────────────────┘      └──────┬───────┘ │
└──────────────────────────────────────────────────────────┼─────────┘
                                                           │ 페이지 이동
┌──────────────────────────────────────────────────────────┼─────────┐
│                   ICAO PKD 동기화 페이지                   ▼         │
│                                                                     │
│  Step 4                         Step 5                              │
│  ┌──────────────────┐          ┌──────────────────┐                 │
│  │ TLS 연결 테스트    │   ──→   │  수동 동기화 실행  │                 │
│  │ (암호화 통신 확인) │          │ (인증서 다운로드)  │                 │
│  └──────────────────┘          └──────────────────┘                 │
└─────────────────────────────────────────────────────────────────────┘
```

### 산출물 요약

| 단계 | 산출물 | 파일 형식 | 설명 |
|------|--------|----------|------|
| Step 1 | CSR PEM | `-----BEGIN CERTIFICATE REQUEST-----` | CA에 제출할 서명 요청서 (인증서 아님) |
| Step 2 | 클라이언트 인증서 | `-----BEGIN CERTIFICATE-----` | CA가 서명한 TLS 인증서 (client.pem) |
| Step 2 | 개인키 | client-key.pem | CSR 생성 시 함께 생성된 개인키 |
| Step 2 | CA 인증서 | ca.pem | 서버 인증서 검증용 CA 공개 인증서 |

## 2. 사전 조건

### 필수 사항

| 항목 | 확인 방법 |
|------|----------|
| **admin 계정 로그인** | 좌측 사이드바에 "시스템 관리" 섹션이 보이는지 확인 |
| **Private CA 설치** | 서버 관리자에게 확인 (`.docker-data/ssl/ca.key`, `ca.crt` 존재) |
| **ICAO PKD LDAP 서버 가동** | ICAO PKD 동기화 페이지에서 "비활성화" 경고가 없는지 확인 |

### ICAO PKD LDAP 서버 인증서

ICAO PKD LDAP 서버(636 포트, LDAPS)에는 **서버 인증서**가 이미 설치되어 있어야 합니다. 이 매뉴얼에서 생성하는 것은 **클라이언트 인증서**로, PKD Relay가 ICAO PKD LDAP 서버에 자신을 인증하는 데 사용합니다.

```
ICAO PKD LDAP 서버                    PKD Relay (우리 시스템)
┌─────────────────┐                  ┌─────────────────┐
│ 서버 인증서       │  ◄── TLS 연결 ──►│ 클라이언트 인증서  │
│ (ldap-server.crt)│    상호 인증      │ (client.pem)     │
│ (사전 설치 완료)  │                  │ (이 매뉴얼에서 발급)│
└─────────────────┘                  └─────────────────┘
```

## 3. Step 1: CSR(인증서 서명 요청) 생성

> **CSR이란?** 인증서가 아닙니다. CA(인증기관)에 제출하여 인증서 발급을 요청하는 **서명 요청서**입니다.

### 3.1 CSR 관리 페이지 접속

사이드바 → **시스템 관리** → **CSR 관리** 클릭

### 3.2 CSR 생성

1. 페이지 우측 상단 **"CSR 생성"** 버튼 클릭

2. 다이얼로그에 정보 입력:

   | 필드 | 입력 예시 | 설명 |
   |------|----------|------|
   | 국가코드 (C) | `KR` | ISO 3166-1 alpha-2 (최대 2자) |
   | 조직명 (O) | `SMARTCORE Inc.` | 기관명 |
   | 일반명 (CN) | `pkd-relay-client` | 클라이언트 식별 이름 |
   | 메모 (선택) | `ICAO PKD TLS 인증서` | 관리용 메모 |

3. 알고리즘 사양 확인 (자동 표시, 변경 불가):
   - **공개키**: RSA 2048 bit
   - **서명**: SHA256withRSA
   - **인코딩**: Base64 (PEM)

4. **"생성"** 버튼 클릭

### 3.3 결과 확인

- 생성된 CSR PEM이 화면에 표시됩니다
- **복사** 또는 **다운로드** 가능 (ICAO PKD에 제출 시 사용)
- CSR 목록에 `CREATED` 상태로 추가됨
- 개인키는 AES-256-GCM으로 암호화되어 DB에 안전하게 저장됩니다

> **참고**: 실제 ICAO PKD 운영 시에는 이 CSR PEM 파일을 ICAO에 제출합니다. ICAO가 서명하여 인증서를 발급해주면 "ICAO 발급 인증서 등록" 버튼으로 등록합니다. 시뮬레이션 환경에서는 다음 단계에서 Private CA로 직접 발급합니다.

## 4. Step 2: TLS 클라이언트 인증서 발급

> **이 단계는 시뮬레이션/테스트 환경 전용입니다.** 실제 운영 시에는 ICAO PKD가 인증서를 발급합니다.

### 4.1 CSR 상세 보기

CSR 목록에서 방금 생성한 항목의 **👁 (상세보기)** 아이콘 클릭

### 4.2 TLS 클라이언트 인증서 발급

1. 상세 다이얼로그 하단의 **"TLS 클라이언트 인증서 발급"** (파란색 버튼) 클릭

2. 확인 메시지 표시:
   > *"Private CA로 TLS 클라이언트 인증서를 발급하시겠습니까?*
   > *발급된 인증서는 ICAO PKD LDAP TLS 상호 인증에 사용됩니다."*

3. **"확인"** 클릭

### 4.3 발급 결과

발급이 완료되면:

- **토스트 알림**: "인증서 발급 완료" + Subject DN + TLS 파일 경로
- CSR 상태: `CREATED` → **`ISSUED`** 변경
- 상세 보기에 발급 인증서 정보 표시:
  - 발급자 (Issuer): Private CA의 DN
  - 시리얼 번호
  - 유효 기간: 발급일 ~ 365일 후

### 4.4 자동 생성되는 TLS 파일

서버 내부에 다음 3개 파일이 자동 저장됩니다:

| 파일 | 용도 |
|------|------|
| `client.pem` | 발급된 클라이언트 인증서 (PKD Relay 신원 증명) |
| `client-key.pem` | 클라이언트 개인키 (TLS 핸드셰이크 서명) |
| `ca.pem` | CA 인증서 (ICAO PKD LDAP 서버 인증서 검증) |

> **별도 파일 복사 불필요**: PKD Relay가 자동으로 이 파일들을 참조합니다.

## 5. Step 3: ICAO PKD 연결 적용

### 5.1 연결 적용

1. CSR 상세 다이얼로그에서 **"ICAO PKD 연결 적용"** (초록색 버튼, `ISSUED` 상태일 때만 표시) 클릭

2. 토스트 알림 표시:
   > *"TLS 인증서가 적용되었습니다. ICAO PKD 동기화 페이지에서 연결 테스트를 수행하세요."*

3. **ICAO PKD 동기화 페이지**(`/sync/icao-ldap`)로 자동 이동됩니다

> **참고**: TLS 인증서 파일은 Step 2에서 이미 서버에 저장되어 있으므로, 이 버튼은 페이지 이동만 수행합니다. PKD Relay 서비스 재시작은 필요 없습니다.

ICAO PKD 동기화 페이지 상단의 **"연결 테스트"** 버튼 클릭

### 6.2 결과 확인

#### 성공 시

| 항목 | 표시 예시 |
|------|----------|
| 상태 | ✅ 연결 성공 |
| 서버 | `icao-pkd-ldap:636` |

| 인증서 수 | `81` (ICAO PKD에 등록된 인증서) |
| TLS 모드 | `TLS / Simple Bind` |

#### 실패 시

| 항목 | 표시 예시 | 조치 |
|------|----------|------|
| 상태 | ❌ 연결 실패 | |
| 에러 | `Connection failed to icao-pkd-ldap:636` | TLS 인증서 재발급 또는 서버 상태 확인 |
| 에러 | `비활성화 상태` | `.env`에서 `ICAO_LDAP_SYNC_ENABLED=true` 확인 |

## 8. 용어 설명

| 용어 | 설명 |
|------|------|
| **CSR** | Certificate Signing Request. 인증서 서명 요청서. CA에 제출하여 인증서 발급을 요청하는 문서. 인증서가 아님. |
| **Private CA** | 내부 인증기관. 자체적으로 인증서를 발급할 수 있는 CA. 시뮬레이션/테스트 환경에서 ICAO CA 역할 대행. |
| **클라이언트 인증서** | PKD Relay가 ICAO PKD LDAP에 자신을 인증하기 위해 제시하는 인증서 (client.pem). |
| **서버 인증서** | ICAO PKD LDAP 서버가 클라이언트에게 자신을 인증하기 위해 제시하는 인증서 (ldap-server.crt). |
| **CA 인증서** | CA의 공개 인증서 (ca.pem). 상대방 인증서의 서명을 검증하는 데 사용. |
| **TLS** | Transport Layer Security. 네트워크 통신을 암호화하는 프로토콜. |
| **TLS 상호 인증** | 서버와 클라이언트가 서로의 인증서를 검증하는 방식. LDAPS:636 포트 사용. |
| **Simple Bind** | TLS 암호화 채널 위에서 ID/비밀번호로 인증하는 방식. |
| **SASL EXTERNAL** | TLS 핸드셰이크 시 제시한 클라이언트 인증서로 인증하는 방식 (비밀번호 불필요). |

## 9. FAQ

### Q1. CSR과 인증서의 차이는?

**CSR**은 "인증서를 발급해주세요"라는 **요청서**이고, **인증서**는 CA가 그 요청을 승인하여 발급한 **최종 결과물**입니다. CSR에는 공개키와 주체 정보만 포함되어 있고, 인증서에는 CA의 서명과 유효기간 등이 추가됩니다.

### Q2. "TLS 클라이언트 인증서 발급" 버튼이 보이지 않습니다.

CSR 상태가 이미 `ISSUED`(발급 완료) 또는 `REVOKED`(폐기)인 경우 버튼이 표시되지 않습니다. 새 CSR을 생성하거나, 상태를 확인하세요.

### Q3. 인증서 발급 후 PKD Relay를 재시작해야 하나요?

**아닙니다.** PKD Relay는 ICAO PKD LDAP 연결 시마다 TLS 파일을 읽으므로 재시작 없이 즉시 적용됩니다.

### Q4. 연결 테스트에서 "연결 실패"가 표시됩니다.

1. ICAO PKD LDAP 서버가 가동 중인지 확인
2. 서버 636(LDAPS) 포트가 열려 있는지 확인
3. `.env`에서 `ICAO_LDAP_USE_TLS=true`, `ICAO_LDAP_PORT=636` 확인
4. TLS 인증서 파일이 올바르게 생성되었는지 확인 (Step 2 재수행)

### Q5. 실제 ICAO PKD 운영 시 절차가 다른가요?

실제 운영 시에는 Step 2가 달라집니다:

| 단계 | 시뮬레이션 (현재) | 실제 운영 |
|------|-----------------|----------|
| Step 1 | CSR 생성 (동일) | CSR 생성 (동일) |
| Step 2 | **Private CA로 직접 발급** ("TLS 클라이언트 인증서 발급" 버튼) | **CSR PEM을 ICAO에 제출** → ICAO가 인증서 발급 → **"ICAO 발급 인증서 등록"** (주황색 버튼)으로 등록 |
| Step 3~5 | 동일 | 동일 |

### Q6. 인증서 유효기간이 만료되면?

발급된 인증서의 유효기간은 365일입니다. 만료 전에 새 CSR을 생성하고 인증서를 재발급하세요 (Step 1~3 반복).
