> **작성일**: 2026-03-22
> **기준**: 법무부 사업계획서 5대 서버 분산 배포 환경
> **현재 버전**: FASTpass® SPKD v2.39.0

## 1. 현재 상태 vs 목표 환경

### 현재 (단일 서버)

```
단일 서버 (로컬/Production)
  └── Monitoring Service (:8084)
      ├── localhost 컨테이너 헬스체크 (curl /api/health)
      ├── nginx stub_status (접속자 수, TCP 연결)
      ├── DB 연결 상태 (Oracle/PostgreSQL)
      ├── LDAP 연결 상태 (OpenLDAP)
      └── 서비스 응답 시간 측정
```

### 목표 (5대 서버 분산)

```
5대 서버 × 다수 서비스 × 이중화
  ├── PKD서버 #1:  Management + PA + LDAP Primary + Gateway
  ├── PKD서버 #2:  Management(DR) + PA(DR) + LDAP Secondary
  ├── 관리서버:     Oracle + 보안SW
  ├── 중계서버 #1:  PKD Relay (ICAO PKD 연계)
  └── 중계서버 #2:  PKD Relay(DR)
  → 5대 서버, 10+ 서비스, 네트워크 구간별 모니터링 필요
```

## 2. 모니터링 체계 구성

### 2.1 3계층 모니터링 아키텍처

```
┌─────────────────────────────────────────────────────────────┐
│                    모니터링 3계층                               │
│                                                             │
│  [Layer 1] 인프라 모니터링 — Sycros v3 Agent (사업계획서 5식)   │
│  ├── OS 레벨: CPU, Memory, Disk I/O, Network 트래픽           │
│  ├── 프로세스 상태, 포트 리스닝                                 │
│  ├── 보안 이벤트 감사, 로그 무결성 검증                         │
│  ├── 관리자 비활동 세션 관리                                   │
│  └── → 정보관리과(제1 청사) 통합모니터링 연계                   │
│                                                             │
│  [Layer 2] 애플리케이션 모니터링 — Monitoring Service (:8084)   │
│  ├── 서비스 헬스체크 (HTTP /api/health 엔드포인트)              │
│  ├── 서비스 응답 시간, 가용률                                  │
│  ├── DB 연결 풀 상태, LDAP 연결 풀 상태                        │
│  ├── ICAO PKD 연계 상태 (동기화 시간, TLS 인증서 만료)          │
│  └── → 관리 웹 UI 대시보드 + SSE 실시간 알림                   │
│                                                             │
│  [Layer 3] 보안 모니터링 — SecuveTOS + Chakra MAX              │
│  ├── 서버 접근 제어, 불법 접근 탐지                             │
│  ├── DB 접근 감사, SQL 인젝션 탐지                             │
│  ├── 파일 무결성 검증                                         │
│  └── → 보안 관제 연동                                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 3. Monitoring Service 확장 상세

### 3.1 현재 모니터링 항목

| 항목 | 구현 상태 | 비고 |
|------|---------|------|
| 서비스 헬스체크 (localhost) | ✅ 구현됨 | /api/health, /api/pa/health, /api/sync/health |
| nginx 접속자 수 | ✅ 구현됨 | stub_status + access log 고유 IP |
| DB 연결 상태 | ✅ 구현됨 | Oracle/PostgreSQL 연결 체크 |
| LDAP 연결 상태 | ✅ 구현됨 | ldapsearch -s base |
| SSE 실시간 알림 | ✅ 구현됨 | NotificationManager |

### 3.2 확장 필요 항목

| 항목 | 우선순위 | 설명 |
|------|---------|------|
| **원격 서버 헬스체크** | 높음 | 5대 서버의 서비스 엔드포인트를 주기적 HTTP 폴링 |
| **서버별 리소스 수집** | 높음 | CPU, Memory, Disk 사용률 (각 서버 에이전트 또는 SSH) |
| **LDAP 이중화 상태** | 높음 | Primary/Secondary MMR 복제 지연 감지 |
| **ICAO PKD 연계 상태** | 높음 | 마지막 동기화 시간, TLS 인증서 만료일, 연결 성공률 |
| **Oracle 상세 메트릭** | 중간 | 세션 수, 테이블스페이스 사용률, 슬로우 쿼리 |
| **SAN 스토리지 용량** | 중간 | 150TiB 사용률 모니터링 |
| **장애 알림 (이메일/SMS)** | 중간 | 서비스 다운 시 관리자 알림 |
| **DR 전환 상태** | 낮음 | Active/Standby 상태 표시, 전환 이력 |

### 3.3 원격 헬스체크 설정 (환경변수 기반)

```bash
# Monitoring Service 환경변수 (docker-compose / .env)

# 원격 서버 헬스체크 대상 목록
MONITOR_TARGETS=pkd1:8081,pkd1:8082,pkd2:8081,pkd2:8082,relay1:8083,relay2:8083

# 또는 상세 설정
MONITOR_PKD_SERVER_1=10.x.x.1      # PKD서버 #1 IP
MONITOR_PKD_SERVER_2=10.x.x.2      # PKD서버 #2 IP
MONITOR_RELAY_SERVER_1=10.x.x.3    # 중계서버 #1 IP
MONITOR_RELAY_SERVER_2=10.x.x.4    # 중계서버 #2 IP
MONITOR_DB_SERVER=10.x.x.5         # 관리서버 IP

# 체크 주기 (초)
MONITOR_CHECK_INTERVAL=30
MONITOR_TIMEOUT=5

# 장애 알림
MONITOR_ALERT_EMAIL=admin@smartcoreinc.com
MONITOR_ALERT_THRESHOLD=3           # 연속 실패 횟수
```

## 4. Sycros v3 연동 방안

### 4.1 Sycros v3 Agent 역할 (사업계획서 기준)

| 기능 | 설명 |
|------|------|
| **시스템 모니터링** | CPU, Memory, Disk, Network (5~8core 당) |
| **CC인증** | 국가정보원의 CC인증을 필한 제품 (EAL2 이상) |
| **OS 커널 보안** | 버퍼 오버플로우, 웜, 바이러스 방어 |
| **사용자 관리** | 접근 허가 할당/제거, 제약사항 관리 |
| **감사 로그** | 보안이벤트 관련 감사로그 생성/저장 |
| **무결성 검증** | 로그 데이터 및 실행 코드에 대한 자체 무결성 검증 |
| **세션 관리** | 관리자 비활동 기간 이후에 대한 세션 관리 |
| **접근 제어** | SSH, FTP, SFTP, RDP, HTTP(S) 등 원격 프로토콜 |
| **보안 정책** | 보안관리 정책 설정 및 커스터마이징 |

### 4.3 정보관리과 통합모니터링 연계 (ECR-04)

사업계획서 ECR-04에 따라 제1 터미널, 2 터미널의 온습도 모니터링과 함께 **기존 시스템 통합모니터링**에 연계해야 합니다.

```
정보관리과(제1 청사) 시스템 통합모니터링
    │
    ├── Sycros v3 Agent (5대 서버) ──→ OS/인프라 메트릭
    │
    ├── 온습도 모니터링 (T1/T2) ──→ 10인치 디스플레이 알람
    │   ※ 설치장소: T2 보안관리과
    │
    └── FASTpass® SPKD Monitoring Service ──→ 애플리케이션 메트릭
        (API 연동 또는 SNMP/Syslog 연계)
```
